# Deployment Guide - EMPOWERRURAL

This document describes how to deploy the EmpowerRural monorepo to production.

---

## 1. Supabase Database Deployment

1. Create a free account on [Supabase](https://supabase.com/).
2. Create a new project named `EmpowerRural`.
3. Navigate to **SQL Editor** in the side navigation panel.
4. Open a **New Query**, copy the contents of `supabase/schema.sql` into the text editor, and click **Run**. This creates all tables, triggers, and Row-Level Security policies.
5. In another query, copy the contents of `supabase/seed.sql` and click **Run** to load initial regional state names, skills checklists, schemes, and courses.
6. Go to **Project Settings** -> **API** to copy the URL and Anon Key.

---

## 2. Backend API Deployment (Railway / Render)

You can host the Express server on [Railway](https://railway.app/) or [Render](https://render.com/).

### Steps for Railway:
1. Connect your Github Repository to Railway.
2. Select **New Project** -> **GitHub Repo** -> Choose the repository.
3. In settings, change the root directory to `apps/api`.
4. Configure variables in the **Variables** tab:
   * `PORT`: `5000`
   * `NODE_ENV`: `production`
   * `SUPABASE_URL`: `https://your-project.supabase.co`
   * `SUPABASE_ANON_KEY`: `your-anon-key`
   * `SUPABASE_SERVICE_ROLE_KEY`: `your-service-role-key`
   * `GEMINI_API_KEY`: `your-gemini-api-key`
5. Click **Deploy**. Railway will start the server using the `npm run start` script defined inside `apps/api/package.json`.

---

## 3. Frontend Hosting (Vercel)

Deploy the React-Vite web client on [Vercel](https://vercel.com/).

1. Sign up on Vercel and click **Add New** -> **Project**.
2. Select your repository.
3. Configure the build parameters:
   * **Framework Preset:** `Vite`
   * **Root Directory:** `apps/web`
   * **Build Command:** `npm run build`
   * **Output Directory:** `dist`
4. Add environment variables:
   * `VITE_API_URL`: Use the domain URL generated by Railway/Render (e.g. `https://empowerrural-api.railway.app`).
5. Click **Deploy**.
6. Set up URL rewrites in a `vercel.json` if needed to route all virtual paths back to `index.html` (Vite's default React Router support):
   ```json
   {
     "rewrites": [
       { "source": "/(.*)", "destination": "/index.html" }
     ]
   }
   ```
